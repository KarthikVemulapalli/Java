Java Architecture (JDK - Java Development Kit):
	1. Compiler
	2. Interpreter
	3. Packages, RunTime Libraries, Java Virtual Machine, JavaAPI

	* Build in keywords are present in JDK
	* Compiler converts JavaProgram (.java file) to '.class file'. The class file contains byte code which is in encrypted format. JVM can only read this '.class' files and execute the written code.
	* Compiler checks the Code Syntax during Compile time (while writing the code).
	* Intrepreter Converts High Level Language to Low Level Language during Runtime (Execution):
		1. Low Level - 0 & 1 (Machine Readable)
		2. High Level - Human Readable
	* With Help of JVM, Libraries and packages - the code execution happens.
	* JVM is OS plaftform dependent, but java is platform independent

	* Java SE is actually a Java Development Kit (JDK) comprises of:
		1. Development Tools
		2. Java Runtime Environment (JRE)
		* Devlopment Tools: This include Java compiler (javac.exe) and Java interpreter (java.exe). These tools are required to prepare a Java application for execution.
		* Java Runtime Environment (JRE):
			This consists of JVM, Java Standard Packages, Runtime Libraries
			Java Virtual Machine (JVM) executes a Java application using certain specific libraries and files provided by JRE. JVM is the actual runtime in which a Java application executes.
		** Note: From JDK 11 onwards there is no Java Runtime Environment (JRE).
		** Note: Development tools like javac and java are available in the bin folder of the JDK installation directory.

Execution process of Java application:
	Source File(.java) -> Compiler (javac.exe) -> Byte Code (.class) -> JVM {Class Loader -> Byte Code Verifier -> Interpreter -> JIT Compiler -> Runtime} -> Operating System -> Hardware

	* javac.exe - Platform independent code generated by compiler
	* Class Loader - Loads all the class files needed for execution
	* Byte Code Verifier - Checks code fragments for illegal code
	* Interpreter - Converts byte code instruction to machine code
	* JIT Compiler - Compiles reusable byte code to machine code
	* Runtime - Assists in overall execution of the program

Coding Best Practices:
	* The name of the class should start with an uppercase letter using Camel Case notation, Example: EmployeeRefer [first alphabet of word is uppercase]
	* The name of methods should start with a lowercase letter and then using CamelCase notation, Example: getName() [first alphabet of first word is lowercase]

Currently Java has 50 Keywords. Ex: class, for, int, public, etc.
	* Primitive datatypes - It holds the value of the data item. There are 8 primitive datatypes: boolean, char, byte, short, int, long, float, double
	* Non-Primitive datatypes - It holds the memory address where the data item (object) is stored. It is also known as Reference datatypes.Ex: Classes, String, enum.

Object Creation and Memory Allocation:
	Ex: Account acct = new Account();
	1. The function of the new keyword is to allocate memory Account object. And assign default values to the member variables.
	2. The reference variable 'acct' points to the Account object created in the memory. Just after memory allocation for the object, Account() constructor is called.

	* A reference variable stores the logical address of an object, but the explicit use of an address is not supported in Java.
	* Unlike in C, pointer manipulation is not possible in Java.
	* The default value remains null for the reference variable till we assign an object to it.

Variables: Local Variable, Static Variable, Instance Variable, Reference Variable
	1. Local Variable - Variables are declared and used inside the same method only.
	2. Static Variable - Static variables are declared directly inside a class and can be used outside the class with the class name (Using Class Object is not preferrable)
	3. Instance Variable - Instance variables (NonStatic) are declared inside the class and Outside the method. And can be used outside class with the class object only
	4. Reference Variable - Variable which stores the address of the object (class)

Type Casting: Java is a strongly-typed language, where type-checking is strictly enforced at runtime.
	1. Widening conversion / Implicit type casting: Converting a variable of smaller datatype to the larger datatype, without data loss is known as widening conversion.
	2. Narrowing conversion / Explicit type casting: Converting a variable of larger datatype to the smaller datatype, with some data loss is known as narrowing conversion.

Datatypes Order: For type casting,
	boolean (1 bit)
	byte(1 byte) -> short(2 byte) -> char(2byte) -> int(4 byte) -> long(8 byte) -> float(4byte) -> double(8 byte)


Java OOPS Concepts:
Polymorphism :
	Types of Polymorphism in Java
	1. Static Polymorphism
		1. Method overloading
		2. It uses compile-time binding
		3. Method invoked is based on the reference type
	2. Dynamic Polymorphism
		1. Method overriding
		2. It uses late binding or runtime binding
		3. Method invoked is based on the object type
